# Multi-Game Prize Pools Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the single hard-coded Lotto contract and UI with a five-game lottery DApp that supports independent rounds, independent prize pools, multi-tier prize distribution, Chainlink VRF draws, and lightweight GSAP interaction feedback.

**Architecture:** The Solidity contract becomes a multi-game engine keyed by `gameType`, with game-specific ticket validation, draw generation, tier resolution, winner registration, and prize accounting. The React frontend moves from fixed front/back selectors to game-driven definitions and dynamic pickers. GSAP is used only for scoped React UI feedback using transforms and `autoAlpha`.

**Tech Stack:** Solidity 0.8.24, Hardhat 3, Chainlink VRF v2.5, React/Vite, ethers v6, GSAP, `@gsap/react`.

```javascript
export function togglePick(selected, value, area) {
  if (selected.includes(value)) {
    const index = selected.indexOf(value);
    return selected.filter((_, i) => i !== index);
  }
  if (selected.length >= area.pick) return selected;
  const next = [...selected, value];
  return area.ordered ? next : sortedNumbers(next);
}
```

For repeatable ordered digits, repeated values must be allowed.

- [ ] **Step 3: Add random pick helper**

```javascript
export function randomPickForArea(area) {
  const values = [];
  while (values.length < area.pick) {
    const next = Math.floor(Math.random() * (area.max - area.min + 1)) + area.min;
    if (area.allowRepeat || !values.includes(next)) values.push(next);
  }
  return area.ordered ? values : sortedNumbers(values);
}
```
